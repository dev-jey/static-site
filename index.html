Introduction

Static site generators are responsible for the making of intermediary fully coded static sites and sites made with content management applications.
Static site generators apply data and content to static web pages and hence generate dynamic views of the webpage.

Most modern static site generators normally include an asset pipeline. The pipeline is responsible for handling a number of processes including:

  1. Asset compilation
  2. Minification of assets
  3. Transpilation
  4. Bundling

Examples of static generators include:

  • Next.js
  • Hugo
  • Gatsby
  • Nuxt
  • Slate
  • Docsify

Must have skills

Before venturing into website development using static site genereator, there are a number of skills required.
One should have a basic understanding of website development with the prequisite knowledge in html and basic javascript.
It is worthwhile to not that most of the modern static site generators are based on javascript.
Here is a list of necessary prerequisite skills:

  • Essentials of Javascript
  • A clear understanding of templates in web development
  • HTML
  • An understanding of how static site generators work

With all these skills, it would be very easy for one to start developing web applications using static site generators

How to implement

To implement static site generation, we are going to use Next.js for this example.
What is Next.js?
>> Next.js is a minimalistic framework for server-rendered React applications as well as statically exported React apps.

First, create a new folder Run

mkdir testfolder

Then run

cd testfolder

Install next.js

npm install --save next react react-dom

Next, add the following build script in the package.json


     { "scripts": { "dev": "next", "build": "next build", "start": "next start" } }

Try out some code, then run the folloqing command to start the server

npm run dev

Pros and cons


Pros

  1. Static site generators play a veryv significant role in improving the speed of a site
  2. Since the developed site is static, there is easy and straightforward integration with CDNs. This saves on time
  3. It is hard to ignore the fact that statically generated sites are simple to develop and maintain
  4. There are additional security benefits that accompany static site generation
  5. Also, statically generated sites are usually easy to scale

Cons

  1. Since they are file driven, their flexibility is limited
  2. Everytime you eant to make updates on the system, you have to redeploy the whole system, which is inconvenient
  3. Static sites have limited features as compared to dynamic sites

Additional tips

To successfully implement a site in a static site generator, here are some tips to keep in mind

  1. Differenciate between a CMS and an SSG
  2. Acknowledge the different security threats and vulnerabilities associated with SSGs
  3. Know how to work with version control software such as git
  4. Get acquainted with the JAM stack (Javascript, APIs, Markup)
